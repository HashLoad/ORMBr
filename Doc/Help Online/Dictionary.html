Dictionary

Mapeamento ›› Atributos ›› Atributos de Proriedades ››

Descrição :

O atributo Dictionary, define para o ORMBr informações sobre cada coluna da tabelas, é ele quem será usado para configurar todas as colunas da tabela no DataSet.

Ao ser instânciado o ORMBr irá criar no DataSet em runtima todos os TFields referente a tabela selecionada, e parametrizar cada um conforme descrito no atributo Dictionary, fazendo assim que cada TField tenha informações necessárias e caracteristicas de cada coluna.


Parametros :

DisplayLabel

Descrição que será mostrado nos componentes Datawere, que podem identificar o que representa a informação.

DefaultExpression

Valor padrão a ser alimentado ao campo, toda vez que for inserir um registro novo, evitando assim que alguma coluna vá sem infromação para a tabela no banco de dados ou deixe a coluna = null.


Definindo 'Date' a coluna será preenchida com a data atual do sistema operacional.

ConstraintErrorMessage

Mensagem de alerta quando houver a validação de rejeição de uma informação na coluna.

DisplayFormat

Define-se como os dados serão mostrados nos componentes Dataware para o usuário final.

EditMask

Define-se um formato dos dados que serão digitados nos componentes Dataware pelo o usuário final.

Alignment

Define-se a posição que o dado será posicionado no componente Dataware na tela para o usuário final.


Exemplos :

[Dictionary('updatedate', 'Mensagem de validação', 'Date', '', '!##/##/####;1;_', taCenter)]

property updatedate: TDateTime Index 0 read Fupdatedate write Fupdatedate;


[Dictionary('Preço Unitário', 'Mensagem de validação', '0','#,###,##0.00', '', taRightJustify)]

property price: Double Index 1 read Fprice write Fprice;

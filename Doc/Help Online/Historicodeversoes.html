Histórico de versões

versão 1.7.1 (Jul 2017)

  • Novo - Adicionado recurso para uso de Enumeration nos modelos, novo Atributo Enumeration().
  • Melhorias - Refatoração.

versão 1.7.0 (Jul 2017)

  • Novo - Criado Driver DataSnap para acesso a dados como o ORMBr, simplesmente facil de usar, como se tivese o C/S, demo na pasta Demo\DataSnap a App Server e App Client
  • Melhorias - Refatorações para o Driver DataSnap
  • Melhorias - Removida a exigência de colocar a palavra reservada INDEX ? nas propriedades do modelo.
  • Molhorias - Rataforações, Compilação de todos os Demos para verificação se com as alterações continuam compilando 100%.

versão 1.6.2 (Jun 2017)

  • Novo - Novo atributo Cascade, para auto gerenciamento de execução de comandos em cascata, quando se usa ObjectSet.
  • Melhorias - Muita rafatoração de código sempre, para melhorar principalmente peformance
  • Novo - Implementação mostrando o uso por completo do ORMBr em uma aplicação Datasnap.

versão 1.6.1 (Jun 2017)

  • Correção - BUG na ordem dos parametros no comando Update em todos os Drivers, motivo que ocasionava a não gravação de multiplos registros alteredos.
  • Correção - BUG não tava gravando conteudo do campo BLOB
  • Correção - Sintaxe do comando SQL para Select no Oracle ao ter paginação trazia a coluna ROWINI como 1a da lista, tive que mudar para o último da lista.
  • Correção - Ajuste na unit ormbr.types.blob.pas para compilar no FMX
  • Correção - Compatibilidade com o D2010 unit ormbr.types.blob.pas
  • Novo - Adicionado diretiva de compilação FMX no ormbr.inc para compilarção com Firemonkey, por default fica desabilitado {.$DEFINE FMX}, para habilitar remova o ponto do inicio
  • Melhorias - Criado dois novos atributos AggregateField e CalcField
  • Correção - Mudado o nome do diretiva FMX para FML no arquivo ormbr.inc, motivo do Delphi por um motivo não detectado não considerar uma classe do tipo TPersistent no método ResgisterClass()
  • Correçao - Sintaxe de paginação no banco Oracle
  • Melhoria - Atribuido recuso para que a propriedade usado para campo calculado, não receba valor pelo TField pois a mesma receberá via o método Get dela.
  • Novo - Criado propriedade na interface IDBConnction de nome AutoManageConnection valor padrão True, o parametro da total responsabilidade ao desenvlvedor de abrir e fechar a conexão com o banco de dados se seu valor for mudado para False.
  • Correção - Criação de tabela no SQLite não estava considerando a definição AutoInc para criar o campo na tabela como Autoincement
  • Correção - Na execução dos scripts usando FireDAC, ocorria um erro dizendo que não existia um transação aberta, foi feito um checagem antes de chamar os métodos Commit e Rollback para não ocorrer mais a exceção.
  • Melhorias - Refatoração no código.

versão 1.6.0 (Jun 2017)

  • Correção - BUG no atributo Association() que não carregava a classe com os dados do ResultSet, quando era para trazer todos os campos da tabela.
  • Melhoria - Refatoração e implementações de novas regras de checagem

versão 1.5.1 (Mai 2017)

  • Melhoria - Descontinuado os métodos OpenWhere e FindWhere foi substituído por Open() e Find(), nesses é possivél passar o filtro "WHERE" e a ordem ORDER BY.
  • Melhoria - Refatoração em toda comada de metadada
  • Novo - Criado driver Oracle.
  • Correção - BUG que não estava trazendo os dados ta tabela com relacionamento OneToOne.
  • Correção - BUG nos comando Update, Insert e Delete usando FireDAC com SQLite, se colocar o nome da tabela antes do campo "Table.Column" da erro, foi removido o nome da tabela deixando somente o nome da coluna.
  • Novo - Criado a interface IDatabaseCompare, ela pode ser instânciada para comparar matedata do database usando Model x Database ou Database x Database.
  • Novo - O ORMBr agora tem um tipo nativo, TBlob para trabalhar com campo BLOB.
  • Correção - Erro de TypeCast ao pegar o valor de um propriedade com Nullable<>.
  • Correção - Tipos de campos no Oracle para comparação do metadata.
  • Melhoria - Refatoração, removendo class singleton e a unit ormbr.types.fields.pas
  • Melhoria - Adicionado no ContainerDataSet os métodos Find do ContainerObjectSet, finalidade para quem usa DataSet poder usar Object sem precisar instânciar um ContainerObjectSet para isso.
  • Melhoria - Alterado atributos Association() e ForeignKey() para atender a multicolunas, pois estava aceitando relacionamento somente por um campo.
  • Novo - Criado um atributo OrderBy(), para que o select feito pelo ORMBr internamente possa ser ordenado com base no atributo.
  • Correção - Paginação no banco de dados MSSQL.

versão 1.3.4 (Abr 2017)

  • Correção - [DCC Fatal Error] ormbr.bind.base.pas(991): F2084 Internal Error: URW1111, no Delphi 2010.
  • Melhoria - Refatoração sempre.
  • Melhoria - Assinatura de Interface para todas as class Singleton.
  • Melhoria - Removida propriedades das classes de mapeamento de objectos e substituido pelo attributo correspondente, o qual tem todas informações passadas no atributo usado no modelo.
  • Melhoria - Refatoração na Unit "ormbr.json.pas", com base na classe de gerar JSON do mORMot Framework, fiz a conversão para nova RTTi, para que o ORMBr consiga ler informações da classe na declaração Public, pois o mORMot usa RTTi antiga que só enxerga em Published.

Ela foi escolhida após bastante pesquisa na internet, onde encontrei o artigo New sample for JSON performance: mORMot vs SuperObject/XSuperObject/dwsJSON/DBXJSON, escrito pelo Autor do mORMot.

  • Novo - Units "ormbr.rest.json.pas" para REST Full e "ormbr.jsonutils.datasnap.pas" para uso com DataSnap.
  • Novo - Demo simples usando recurso de JSON e REST Full com Datasnap, está na pasta "..\ORMBr\Demo\Datasnap".
  • Novo - Driver ADO para acesso a dados.
  • Novo - Demo mostrando o basico de como usar o ORMBr com Datasnap.
  • Novo - Demo trabalhando como JSON no ORMBr.
  • Correção - Método NextPacket, agora deve passar o parâmetro do tipo TObjectList<T> que receberá os registros do proxímo pacote internamente.

versão 1.3.3 (Abr 2017)

  • Melhoria - Implementado recurso para atualizar dados da tabela externa com relacionamento OneToOne quando a coluna chave de relacionamento for mudada.
  • Melhoria - Otimização no método Open(), para que o SELECT seja construído somente uma vez.
  • Melhoria - Refatoração nos métodos Open() e Find(), para aceitarem o parâmetro ASQL como String.
  • Melhoria - Renomeado método GetNextPacket para NextPacket.
  • Melhoria - Liberado o método NextPacket, para ser chamado pela aplicação, caso a propriedade AutoNextPacket = False;
  • Melhoria - Refatoração nos métodos DisableDataSetEvents e EnableDataSetEvents.
  • Melhoria - Grande refatoração para aumento de performance, passando a usar mais informações mapeadas do objeto e menos RTTI.
  • Correção - Ajuste feito para respeitar a paginação de registros, quando passado um comando SELECT pelo método Open() e Find() direto, não estava sendo feito paginação nenhuma.
  • Novo - Atualização do demo que não usa dataset, mostrando como criar um campo Aggregate, direto na class totalizando determinado campo.
  • Novo - Criado método FindWhere(AWhere: string), esse método adiciona a clausula WHERE ao final do comando SELECT montado, passando assim um filtro para restringir o retorno de registros do banco de dados em um TObjectList<M>
  • Novo - Criado método OpenWhere(AWhere: string), esse método adiciona a clausula WHERE ao final do comando SELECT montado, passando assim um filtro para restringir o retorno de registros do banco de dados em um IDBResultSet.
  • Novo - Criado a propriedade AutoNextPacket, para que o método NextPacket possa ser controlado pela aplicação, valor padrão é TRUE;
  • Novo - Criado interface ICommandMonitor que irá monitorar todos os comandos executado pelo ORMBr e mostrar em uma tela o que foi executado. Funcionalidade mostrada no demo na pasta ..\ORMBr\Demo\Data\FireDAC.
  • Novo - Driver TFactorySQLite para acesso nativo ao SQLite, o driver usa componente de terceiro, as units agora fazem parte do projeto ORMBr e está na pasta ..\ORMBr\Source\External\SQLite, podem ser usadas sem precisar instalar o componente na IDE, basta instânciar o componente direto na aplicação.

Exemplo :

 FDatabase := TSQLiteDatabase.Create(Self);

 FDatabase.Filename := '..\Database\database.db3';

versão 1.3.1 (Abr 2017)

  • Novo - Criado método Modify(AObject: M) clona o objeto no seu estado atual, para no Update() identificar o que foi mudado.

Exemplo :

var

oMasterUpd: Tmaster;

begin

 oMasterUpd := Tmaster.Create;

 try

   // Faz uma cópia do objeto antes dele sofre as mudanças necessárias

   oMaster.Modify(oMasterUpd);

   oMasterUpd.master_id := 1;

   oMasterUpd.description := 'Nome alterado';

   // Altera o registro no Banco

   oMaster.Update(oMasterUpd);

 finally

   oMasterUpd.Free;

 end;
